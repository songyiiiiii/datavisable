# Nyx 数据集描述 — 统一技术文档

> 本文档为任务一至任务四的共享数据描述，各任务报告引用此文档，不再重复描述数据处理细节。

## 一、数据来源

Nyx 是一款基于 AMReX 框架开发的并行自适应网格细化（AMR）宇宙学模拟代码，以可压缩流体力学与暗物质 N 体模拟为核心，用于模拟宇宙演化过程。模拟产生的低密度气体填充着星系间的巨大空隙，是理解暗能量、暗物质和宇宙热演化历史的主要观测手段。

本数据集为 Nyx 模拟在固定均匀网格上输出的气体密度场快照，共计 100 个时间步，覆盖从早期均匀态到晚期团块化态的完整演化过程。

## 二、数据格式

### 2.1 文件组织

| 属性 | 值 |
|------|-----|
| 文件数量 | 100 个（`0000.dat` – `0099.dat`） |
| 单文件大小 | 8,388,608 字节（8 MiB） |
| 总数据量 | ~800 MiB |
| 命名规则 | `%04d.dat`（4 位零填充序号） |

### 2.2 二进制编码

| 属性 | 值 |
|------|-----|
| 数据类型 | `float32`（单精度浮点，IEEE 754） |
| 字节序 | 小端（little-endian） |
| 物理量 | 气体密度（自然对数尺度，ln ρ） |
| 数值范围 | 约 7.75 – 14.45（跨越约 3 个数量级线性空间） |

### 2.3 内存布局

数据按**列优先顺序（Fortran 顺序）**存储：先 Z 轴，再 Y 轴，最后 X 轴。即对于索引 `(ix, iy, iz)`，其在文件中的线性位置为：

$$ \text{offset} = iz + iy \times N_z + ix \times N_y \times N_z $$

其中 $N_x = N_y = N_z = 128$。

### 2.4 读取规范

```python
import numpy as np

raw = np.fromfile(filepath, dtype="<f4")            # 小端 float32
data_3d = raw.reshape((128, 128, 128), order='F')    # Fortran 列优先
# 结果索引顺序: data_3d[z, y, x]
```

**关键风险点**：若误用 `order='C'`（C 行优先），将导致 X/Y/Z 轴数据严重错位，三维结构完全失真。PyVista/VTK 期望 C 顺序，使用前需转置：

```python
data_xyz = np.ascontiguousarray(np.transpose(data_3d, (2, 1, 0)))
# 结果索引顺序: data_xyz[x, y, z]
```

## 四、处理原则

**原则一：数据保真优先**。读取时严格保持原始字节序（little-endian）和数据精度（float32），不进行任何有损压缩或降采样。可视化输出忠实反映模拟的物理量值。

**原则二：内存-速度权衡**。完整数据集 100 步 × 128³ × 4 字节 ≈ 800MB。Dash 服务启动时预加载全部 100 步用于 TimeWheel 统计计算（~800MB 常驻），Task 1 图片预渲染 51 帧存储在磁盘（每帧约 400KB PNG）。Task 4 的 3D 交互视图采用 4× 子采样（128→32，32,768 点）。

**原则三：统一参数基准**。所有时间步的可视化使用完全相同的参数（传递函数、相机、光照），确保帧间视觉差异仅来源于底层物理密度场的演化。

**原则四：感知均匀的色彩映射**。全项目统一采用 blue→gray→red 渐变色板（近似 CIELAB 感知均匀）。蓝色 = 低密度空洞（`#1A4A7A` / `#2166AC`），灰色 = 中间密度纤维网（`#666666` / `#999999`），红色 = 高密度团簇（`#B2182B` / `#8B1A2B`）。全部图表使用纯白背景（`#FFFFFF`），避免使用 jet/rainbow 等非均匀色板。

**原则五：可复现性**。所有随机采样固定种子值 `rng = np.random.default_rng(42)`，确保每次运行产生完全一致的子采样结果。
